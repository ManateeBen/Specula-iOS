# Specula Backlog

## P1 - EPUB 代码块显示不全且横向拖动触发翻页

**状态：** 待处理  
**记录时间：** 2026-07-18  
**影响范围：** EPUB 阅读页，技术书与包含长代码行的章节

### 问题

代码块按原始行宽渲染时，内容超出手机屏幕，需要横向拖动才能查看完整代码。阅读器同时使用左右滑动进行章节翻页，导致代码块横向滚动与翻页手势冲突，用户容易在查看代码时误翻章。

### 推荐方案

1. 代码块默认启用软换行，在手机宽度内完整显示，优先消除横向手势冲突。
2. 每个代码块提供“换行 / 原样”切换；仅在原样模式启用横向滚动。
3. 原样模式下，手势起点位于代码块区域时，由代码块消费横向拖动，阅读器不得翻页。
4. 使用等宽字体，并允许代码字号比正文小 1-2 级。

### 后续增强

- 超过 15 行的代码块默认折叠，并提供展开/收起。
- 增加复制代码操作。
- 根据语言信息增加语法高亮；无法识别语言时保持纯文本显示。

### 验收标准

- 默认模式下，长代码行无需横向拖动即可完整阅读。
- 原样模式可横向滚动，拖动代码块不会触发上一章或下一章。
- 代码块外左右滑动翻页行为保持不变。
- 中英文、长 URL、无空格字符串和缩进代码均不会撑破页面宽度。
- 切换章节、退出应用后不会出现空白代码块或阅读位置异常。

### 备注

PDF 保留原版面渲染，不纳入本问题；PDF 的缩放与翻页手势冲突应单独评估。
